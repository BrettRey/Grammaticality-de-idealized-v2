# Collapsed Counterpart Experiment Protocol

**Date:** 2026-06-09
**Status:** Protocol and queue registered; no graph scores or labels changed.
**Queue:** `data/module-compression/collapsed-counterpart-queue.csv`

## Problem

The workbench is discovery-first, but its controlled ontology is stratified: it keeps constructs
such as reported acceptability, grammaticality attribution, community licensing, processing cost,
production probability, ideology, and repair pressure apart.

That separation is a theoretical wager. It should not win merely because the ontology makes
collapsed rivals look crude.

## Rule

For each scoped module, build the strongest minimal collapsed counterpart that deliberately merges
the module's distinctive constructs into a simpler rival. Mark the collapse as a wager rather than
as a lint violation, then run the collapsed graph on the same authorizing and held-out evaluations.

A stratified module earns pressure only where it predicts or explains a contrast that the collapsed
counterpart cannot match without post-hoc exceptions.

## Current State

`AGR` now has the first registered collapsed counterpart:

- stratified module: `agreement-controller-override-candidate`;
- collapsed rival: `agreement-construction-listing-baseline-candidate`;
- live discriminator: scout-selected low-frequency QN-like frames before local item preferences are
  known.

The remaining scoped modules need counterpart rivals before any general claim that the stratified
ontology has discovered kinds rather than vocabulary.

## Pass/Fail Boundary

The stratified ontology gains support if collapsed counterparts repeatedly fail on held-out or
data-bearing contrast cells where the stratified modules survive without adding post-hoc local
exceptions.

The stratified ontology loses pressure if collapsed counterparts match the scoped modules across
held-out/data-bearing cells with fewer constructs and no additional exceptions. In that case, the
workbench should merge, demote, or relabel the relevant constructs rather than protect the existing
ontology.
