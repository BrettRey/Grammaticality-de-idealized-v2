# Phenomenon Card: unbounded dependency licensing

**Stress test:** Dependency licensing, recoverability, and preemption.

## Description

Unbounded dependencies make it easy to recover intended meanings from forms that are nevertheless
not licensed in the relevant construction. They also create contrast cells where a competing
licensed dependency pattern repeatedly occupies the same functional opportunity.

## Source Pointers

- Source IDs: `cgel-unbounded-dependencies`
- Related seed card: [left-branch-extraction](left-branch-extraction.md)

## Why It Matters

The case is a clean stress test for recoverability without licensing, and for preemption without
raw-frequency reduction.

## Minimum Contrast Cells

- ordinary wh dependency with a licensed gap;
- recoverable but non-licensed dependency configuration;
- high-opportunity environment with a stable competitor construction;
- low-opportunity environment where absence is less probative.

## Candidate Nodes

- operator_value

## Graph Failure Modes

- Treats dependency recoverability as sufficient for licensing.
- Treats absence from corpora as probative without opportunity mass.
- Ignores licensed competitors in the same functional slot.
- Treats all extraction failures as the same mechanism.

## Predicted Discriminators

- An operator-gap graph should predict persistent rejection where recoverability is high but
  preemption is also high.
- A processing graph should not be allowed to explain the case only with difficulty, since some
  rejected variants are short and intelligible.
